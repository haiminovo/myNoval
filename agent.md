# Agent 工作规范

本文档用于约束后续写作、改稿、整理资料时的同步规则。目标是让设定、剧情、时间线、角色卡和正文保持一致，避免同一信息在多个文件里各说各话。

## 项目架构

### 文档分层

1. `世界观/小说圣经.md`
   - 世界规则、种族、势力、能力体系、核心悬念的最高权威。
   - 凡涉及世界底层规则、因果律、以太、王座、神明、教派机制、地理与能力体系，以此文件为准。

2. `剧情/大纲.md`
   - 剧情走向、卷结构、章节目标、钩子、反转和主线推进的最高权威。
   - 凡涉及“接下来发生什么”“某章承担什么功能”“角色在剧情中何时知道/行动/转折”，以此文件为准。

3. `正文/myNoval.md`
   - 已经落地成文的叙事事实。
   - 正文不能随意被辅助资料覆盖。若正文与大纲冲突，先判断是大纲过期、正文偏航，还是用户明确要求重写；未经确认，不直接改动正文核心事实。

4. `剧情/时间线.md`
   - 事件顺序、年代、章节日程和因果先后的索引文件。
   - 时间线服务于圣经、大纲和正文，不单独创造新设定或新剧情。

5. `角色/*.md`
   - 角色身份、性格、动机、关系、状态和台词资料。
   - 角色卡服务于圣经和大纲；角色细节可以补充，但不得改写世界规则或主线走向。

6. `README.md`
   - 项目入口说明。
   - 只记录目录和简要说明，不承载权威设定。

## 权威优先级

### 设定类信息

设定冲突时，按以下顺序判断：

1. `世界观/小说圣经.md`
2. 用户本轮明确指令
3. `正文/myNoval.md` 中已成文事实
4. `角色/*.md`
5. `剧情/时间线.md`
6. `剧情/大纲.md` 中的设定性描述
7. `README.md`

原则：小说圣经最权威。任何世界观、能力、势力、种族、王座、以太、观测者、进化教派等设定变更，都必须先写入或同步到 `世界观/小说圣经.md`。

### 剧情类信息

剧情冲突时，按以下顺序判断：

1. `剧情/大纲.md`
2. 用户本轮明确指令
3. `正文/myNoval.md` 中已成文事实
4. `剧情/时间线.md`
5. `角色/*.md`
6. `世界观/小说圣经.md` 中的剧情示例或背景描述
7. `README.md`

原则：剧情走向以大纲为最权威。章节安排、卷目标、钩子、反转、人物登场顺序、主线推进，以 `剧情/大纲.md` 为准。

### 已成文正文的特殊规则

- 正文是“已经发生的叙事”，不是普通草稿索引。
- 如果大纲和正文不一致，先标记为“待同步冲突”，不要默认用大纲覆盖正文。
- 若用户要求“按大纲修正文”，可以改正文。
- 若用户要求“整理资料/同步设定”，优先更新辅助资料，让它们承认正文中已经稳定的事实；但如果正文明显违反小说圣经的底层设定，应先提示冲突点。

## 同步流程

### 修改设定时

1. 先改 `世界观/小说圣经.md`。
2. 检查并同步受影响文件：
   - `角色/*.md`：身份、能力、动机、状态是否受影响。
   - `剧情/大纲.md`：章节钩子、反转、主线解释是否受影响。
   - `剧情/时间线.md`：历史事件和章节事件是否受影响。
   - `正文/myNoval.md`：若已成文内容冲突，列出冲突，不擅自大改正文。
3. 修改后在相关文件末尾更新版本或更新日期，保持格式与原文件一致。

### 修改剧情走向时

1. 先改 `剧情/大纲.md`。
2. 检查并同步受影响文件：
   - `剧情/时间线.md`：事件顺序、章节日程、因果先后。
   - `角色/*.md`：角色状态、关系阶段、动机变化。
   - `世界观/小说圣经.md`：只有当剧情改动引入新世界规则时才同步圣经。
   - `正文/myNoval.md`：仅在用户明确要求改正文时同步正文。
3. 如果剧情改动改变已完结章节，应标记正文修订范围，如“第14-16章需回改”。

### 写新正文时

1. 先查 `剧情/大纲.md`，明确本章目标、钩子、反转和结尾位置。
2. 再查 `世界观/小说圣经.md`，确认本章涉及的能力、势力、地理和底层规则。
3. 查相关 `角色/*.md`，确认人物当前状态、动机和说话方式。
4. 查 `剧情/时间线.md`，确认事件发生顺序和时间跨度。
5. 写入 `正文/myNoval.md` 后，如正文落地了新的稳定事实，反向同步到大纲、时间线或角色卡；若只是描写细节，不需要同步。

### 整理资料时

1. 不把推测写成事实。
2. 不把正文里的临时感受、误判、谎言直接写成客观设定。
3. 区分三种信息：
   - 客观事实：世界真实规则或已经确认的事件。
   - 角色认知：某个角色当时相信的事，可能是错的。
   - 叙事伏笔：作者暂时不公开答案的悬念。

## 冲突处理规范

### 冲突类型

- 设定冲突：能力规则、种族逻辑、势力机制、地理、历史不一致。
- 剧情冲突：章节事件、登场顺序、角色选择、卷目标不一致。
- 时间冲突：日期、天数、前后顺序、年龄、事件间隔不一致。
- 角色冲突：性格、动机、关系、状态、已死亡/存活不一致。
- 叙述冲突：正文已经写出的事实与资料文件不一致。

### 处理顺序

1. 判定冲突属于设定、剧情、时间、角色还是正文落地问题。
2. 按对应权威优先级确定基准文件。
3. 修改低权威文件，使其服从高权威文件。
4. 如果高权威文件之间冲突：
   - `世界观/小说圣经.md` 管设定。
   - `剧情/大纲.md` 管剧情走向。
   - 两者仍无法同时满足时，暂停大范围改动，向用户说明冲突点并给出建议方案。

### 冲突记录格式

遇到暂时不能解决的冲突，在相关文件或工作回复中使用：

```markdown
【待同步冲突】
- 冲突点：
- 涉及文件：
- 当前权威判断：
- 建议处理：
```

## 数据同步检查清单

每次进行跨文件修改后，至少检查以下项目：

- 圣经是否仍是唯一设定源。
- 大纲是否仍是剧情走向源。
- 时间线是否只记录顺序，没有私自新增设定。
- 角色卡是否与角色当前状态一致。
- 正文是否被误改、漏改或与新规则冲突。
- 版本号或更新日期是否需要调整。

## 写作一致性规则

- 不新增与圣经冲突的能力、势力、种族或王座规则。
- 不让角色知道他们尚未知道的信息。
- 不提前兑现大纲中尚未发生的反转。
- 不把读者视角信息写进角色视角。
- 不把“系统提示”写成真正游戏系统，除非大纲或圣经明确改变该设定；现有规则下它是观测者伪装。
- 诺缇相关核心判断必须同时兼顾三层：陈诺意识、427号试验体残留、观测者连接。
- 王座不是单纯力量等级，而是世界承认的规则权限。

## Agent 执行原则

- 开始写作或改稿前，先确认本次任务属于“设定修改”“剧情修改”“正文创作”“资料整理”还是“冲突修复”。
- 能从本地文档判断的，不向用户反复追问。
- 涉及高权威文件改动时，要同步检查低权威文件。
- 只做与任务相关的同步，不进行无关重构。
- 若发现用户指令与现有权威文件冲突，以用户本轮明确指令为当前任务目标，但需要指出会影响哪些权威文件。
- 保持中文写作和项目已有格式风格。
